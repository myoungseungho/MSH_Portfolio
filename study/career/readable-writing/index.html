읽히는 글의 원리

상세페이지부터 기획서까지, 사람을 움직이는 글의 비밀

2025.01 · Q&A 학습 기록

Part 1. 욕망의 본질

"사고 싶다"는 감정이 뭐야? 배고플 때 밥 먹고 싶은 거랑 같아?

배고프면 밥 먹어. 그럼 배고픔이 사라져.

목마르면 물 마셔. 그럼 갈증이 사라져.

근데 "사고 싶다"는? 그거 사면 뭐가 사라져?

만족해서 사라지지 않나?

진짜?

새 폰 샀어. 만족했어.

근데 6개월 뒤에 신형 나왔어.

만족 어디 갔어?

밥 먹고 배부른 건 6개월 뒤에 신형 밥 나왔다고 다시 배고파져?

핵심
배고픔사고 싶다
채우면 사라짐채워도 또 생김
결핍욕망
욕망은 어디서 와?

무인도에 너 혼자 있어. 아무도 없어. 인터넷도 없어. 광고도 없어.

새 폰 사고 싶어?

아니. 필요 없지.

근데 카페 갔어. 옆 사람이 새 폰 꺼내. 화면 크고, 얇고, 카메라 좋아 보여. 그때 네 폰 어때 보여?

초라해 보여.

근데 생각해봐. 네 폰, 뭐가 바뀌었어?

5분 전이랑 똑같은 폰이야. 배터리도 그대로, 화면도 그대로, 기능도 그대로.

근데 갑자기 "초라해 보여."

바뀐 건 폰이 아니야. 비교가 생긴 거야.

핵심
결핍욕망
내 안에서 온다비교에서 온다
채우면 끝비교가 있는 한 끝이 없다

Part 2. 간극의 기술

상세페이지가 뭘 하고 있는 거야?

상세페이지 슥슥 넘기면 "사고 싶다"가 생겨.

그 상세페이지가 뭘 하고 있는 거야?

뭐랑 뭘 비교하게 만들고 있는 거지?

지금의 나랑 저걸 가진 나?

정확해.

상세페이지가 하는 일 "지금의 나" ←――― 간극(GAP) ―――→ "이걸 가진 나" 이 간극이 욕망이야.
이 간극이 너무 크면 어떨 것 같아? 예를 들어, 100억짜리 슈퍼카 상세페이지.

포기하지.

"나랑 상관없는 세계"가 돼버려.

간극의 크기
간극 크기반응결과
간극 없음"이미 있는데 뭐"욕망 없음
간극 적당"아 저거 갖고 싶다"욕망
간극 너무 큼"나랑 상관없는 세계"포기

잘 만든 상세페이지는 "닿을 듯 말 듯한 거리"를 만든다.

Part 3. 도파민과 기대

도파민이 언제 가장 많이 나오는지 알아?

보상 받을 때?

아니.

보상 받으면 도파민 뚝 떨어져.

그럼 언제?

"받을 수 있을 것 같은데, 아직 안 받았을 때"

기대. 설렘. 곧 될 것 같은 느낌.

슬롯머신 생각해봐. 돈 딸 때 기분 좋지. 근데 제일 심장 뛰는 순간?

릴이 돌아가고 있을 때야.

상세페이지 슥슥 넘기고 있어. 아직 안 샀어. 근데 "저거 사면 나도..." 상상하고 있어. 이때 뇌에서 뭐가 터지고 있는 거야?

도파민.

상세페이지는 "기대"를 파는 거야.

도파민의 원리
도파민 = 보상 받을 때 X 도파민 = 보상을 기대할 때 O "닿을 듯 말 듯" = 기대 최대화 = 도파민 폭발
실제 경험 예시

아이디어는 많은데 구현 능력이 없었을 때 → 간극이 너무 큼 → 포기

클로드 코드로 "만들 수 있을 것 같아"가 됐을 때 → 닿을 듯 말 듯 → 도파민 폭발

새벽 3시까지 일해도 안 힘든 이유: 매번 "될 것 같은" 기대가 있으니까.

Part 4. 스토리의 힘

사고 싶은 물건이 있으면 그걸 만든 상세페이지는 어때?

진짜 술술 읽혀. 계속 보고 싶어.

마치 드라마처럼.

"드라마처럼" - 이거 핵심이야. 드라마엔 뭐가 있어?

주인공 있지? 그 주인공한테 문제가 생기지? 그걸 해결하잖아.

드라마 구조: 주인공 → 문제 발생 → 해결 "어떻게 되지?" → 궁금함 → 계속 봄
상세페이지는?
상세페이지 구조: 당신(주인공) → 지금 불편하죠?(문제) → 이거 쓰면 해결됩니다(해결)
딱딱한 보고서는?
보고서: 본 시스템은 → 이러이러한 기능이 → 있습니다 주인공? 없음. 문제? 없음. 그냥 기능 나열.
핵심

뇌는 스토리를 좋아해.

스토리가 없으면 "정보"야. 정보는 에너지가 들어.

스토리가 있으면 "드라마"야. 드라마는 궁금해서 계속 봐.

Part 5. 적용: 읽히는 문서

딱딱한 보고서, 기획서, 기술 문서. 왜 읽기 싫어?

두 가지야:

  • 나랑 상관없어 보여서 → 간극이 안 만들어짐 → 욕망 없음
  • 이해하는데 에너지가 많이 들어서 → 뇌가 거부함
딱딱한 보고서의 주어가 뭐야?

"이 시스템은~", "본 기능은~", "해당 프로세스는~"

상세페이지의 주어는?

"당신은~", "이제 당신도~", "당신의 하루가~"

"독자의 니즈를 알아야 한다" - 이건 너무 뜬구름이야. 구체적으로 뭐야?

이거야:

이 사람이 이 문서를 여는 순간,
머릿속에 뜨는 질문이 뭐지?

예를 들면?
읽는 사람머릿속 질문
개발자가 기획서 열면"내가 뭘 만들어야 돼?"
상사가 보고서 열면"그래서 어쩌라고? 내가 뭘 결정해야 돼?"
유저가 업데이트 공지 열면"이거 하면 나한테 뭐가 좋아?"

그 질문에 답하는 순서로 써. 그게 스토리가 되는 거야.

나쁜 문서 vs 좋은 문서
나쁜 문서: 정보 → 정보 → 정보 → (알아서 파악해) 좋은 문서: 당신의 문제 → 이렇게 해결됨 → 이렇게 하면 됨

총정리: 읽히는 글의 원리

욕망의 본질

  • 결핍은 채우면 사라지지만, 욕망은 채워도 또 생긴다
  • 욕망 = 비교에서 온다
  • 상세페이지는 "비교"를 만든다: 지금의 나 vs 이걸 가진 나

간극의 기술

  • 간극이 없으면 → 관심 없음
  • 간극이 적당하면 → 욕망
  • 간극이 너무 크면 → 포기
  • 핵심: "닿을 듯 말 듯한 거리"

도파민과 기대

  • 도파민 = 보상 받을 때가 아니라, 기대할 때 터진다
  • 상세페이지는 "기대"를 파는 것

스토리의 힘

  • 뇌는 스토리를 좋아한다
  • 드라마 = 주인공 + 문제 + 해결
  • 정보 나열은 에너지가 들고, 스토리는 궁금해서 계속 본다

읽히는 문서의 공식

1. 읽는 사람이 누구야? (주인공) 2. 그 사람이 이걸 여는 순간 머릿속 질문이 뭐야? (문제) 3. 그 질문에 답하는 순서로 써 (해결) 4. 간극을 만들어 (지금 vs 이걸 알면/쓰면) 5. 닿을 듯 말 듯하게 (너무 어렵지 않게)

핵심 통찰

  • 상세페이지든, 기획서든, 보고서든 원리는 같다
  • "읽는 사람의 머릿속 질문"에 답하는 순서로 써라
  • 주어를 "시스템"에서 "당신"으로 바꿔라
  • 정보 나열이 아니라, 스토리로 풀어라
