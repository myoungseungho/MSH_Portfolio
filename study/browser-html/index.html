브라우저와 HTML: 웹의 기초

초보자를 위한 개념 정리 2025.01
브라우저 HTML HTTP HTTPS 웹

개요

웹사이트를 열면 화면에 글자, 이미지, 버튼이 보여요. 이게 어떻게 동작하는 걸까요?

브라우저가 뭔지, HTML이 뭔지, HTTP와 HTTPS는 뭐가 다른지 차근차근 알아봅시다.

1. 브라우저란?

Chrome, Edge, Safari, Firefox... 웹사이트 열 때 쓰는 프로그램이에요.

질문

브라우저에 주소(naver.com) 치면 어떤 일이 일어날까요?

답

서버에서 뭔가를 받아와서 화면에 그려주는 거예요!

브라우저 서버 │ │ │ "네이버 페이지 주세요" ──────────> │ │ │ │ <────────────── HTML 문서 + 이미지 │ │ │ │ 화면에 예쁘게 그림 │

2. 서버가 보내주는 것

서버가 보내주는 건 크게 두 종류예요:

서버가 보내주는 것: ┌─────────────────────────────────┐ │ HTML 문서 (설계도) │ │ "제목은 여기, 이미지는 저기" │ └─────────────────────────────────┘ + ┌─────────┐ ┌─────────┐ │ 이미지1 │ │ 이미지2 │ ... └─────────┘ └─────────┘
파일 역할 비유
HTML (.html) 구조/뼈대 건물 설계도
CSS (.css) 스타일/디자인 인테리어, 페인트
JavaScript (.js) 동작/기능 엘리베이터, 자동문
이미지 (.jpg, .png) 사진 액자, 그림
폰트 (.woff, .ttf) 글씨체 글씨 스타일

웹페이지의 구성

HTML (뼈대) + CSS (옷 입히기) + JavaScript (움직이게 하기) = 완성된 웹페이지!

3. HTML이란?

HyperText Markup Language

  • Markup = 표시하다
  • Language = 언어

즉, "이건 제목이야", "이건 문단이야", "여기에 이미지 넣어"라고 표시해두는 언어예요.

실제 HTML 코드

<h1>제목입니다</h1>
<p>이건 문단이에요</p>
<img src="사진.jpg">
<button>클릭</button>

태그 (Tag)

<h1>, <p>, <img> 이런 걸 태그라고 불러요.

<h1>제목입니다</h1> ↑ ↑ 여는 태그 닫는 태그
태그 의미 결과
<h1> Heading 1 (제목) 큰 제목
<p> Paragraph (문단) 일반 글
<img> Image (이미지) 사진 표시
<a> Anchor (링크) 클릭하면 이동
<button> Button (버튼) 누를 수 있는 버튼
<div> Division (구역) 영역 묶기

4. 브라우저가 하는 일: 렌더링

브라우저가 HTML을 받아서 화면에 그려주는 것을 렌더링(Rendering)이라고 해요.

<h1>안녕하세요</h1> → 안녕하세요 (크고 굵은 글씨) <p>반갑습니다</p> → 반갑습니다 (일반 글씨) <button>클릭</button> → [ 클릭 ] (버튼 모양)

브라우저가 태그를 보고 "아, 이건 제목이니까 크게 그려야지" 하고 판단하는 거예요.

브라우저의 전체 흐름

┌─────────────────────────────────────────────────┐ │ 브라우저 │ │ │ │ 1. 서버한테 HTML 요청 │ │ 2. HTML (+ CSS, JS, 이미지) 받아옴 │ │ 3. HTML 해석 (파싱) │ │ 4. 화면에 그림 (렌더링) │ │ │ └─────────────────────────────────────────────────┘

5. HTTP란?

질문

브라우저가 서버한테 HTML을 요청할 때, 그냥 막 요청하면 안 되잖아요? "이런 형식으로 요청해라" 하는 규칙이 있어야 하지 않을까요?

그게 바로 HTTP예요!

HTTP = HyperText Transfer Protocol

Transfer = 전송

Protocol = 규약/규칙

즉, HTML(HyperText)을 주고받는 규칙이에요!

브라우저 서버 │ │ │ "GET /index.html 주세요" ──────>│ ← HTTP 요청 │ │ │<────── "여기 HTML이요" │ ← HTTP 응답 │ │

6. HTTPS란?

HTTP에는 치명적인 문제가 있었어요.

HTTP로 통신하면: 브라우저 ────── 인터넷 ────── 서버 │ 해커가 보면: "로그인: admin" "비밀번호: 1234" ← 그대로 보임!

암호화가 안 돼 있어서 누가 중간에서 훔쳐볼 수 있었어요!

해결책: HTTPS

HTTPS = HTTP + Secure (SSL/TLS 암호화)

구분 HTTP HTTPS
암호화 X O
포트 80 443
주소창 http:// https:// + 자물쇠
보안 위험 안전
HTTP: 브라우저 ──── (평문) ──── 서버 ← 해커가 볼 수 있음 HTTPS: 브라우저 ──── (암호화) ── 서버 ← 해커가 봐도 모름

7. HTTP vs HTTPS는 누가 정하나?

질문

어떤 사이트는 http://, 어떤 사이트는 https://... 이건 누가 정하는 거예요?

답

서버 측에서 정해요!

서버 관리자가: 1. SSL 인증서를 발급받고 2. 서버에 설치하고 3. "HTTPS로 서비스할게요" 설정 → 그러면 HTTPS로 접속 가능!
상황 결과
서버에 SSL 인증서 없음 HTTP만 가능 (http://)
서버에 SSL 인증서 있음 HTTPS 가능 (https://)
서버가 강제 리다이렉트 설정 HTTP로 접속해도 → HTTPS로 자동 이동

실제 동작 예시

당신: http://naver.com 입력 서버: "어어, HTTP로 왔네? HTTPS로 다시 와!" → https://naver.com 으로 자동 이동시킴 브라우저: https://naver.com 다시 접속

요즘 대부분의 사이트가 강제로 HTTPS로 보내요.

왜 서버가 정하냐면

SSL 인증서는 서버에 설치하는 거거든요.

클라이언트(브라우저)는 그냥 서버가 주는 대로 받는 거예요.

최종 요약

웹의 기초 한눈에 보기

브라우저 = 서버에서 HTML을 받아 화면에 그려주는 프로그램

HTML = 웹페이지의 구조를 정의하는 언어 (+ CSS, JS)

HTTP = HTML을 주고받는 규칙

HTTPS = HTTP + 암호화 (SSL/TLS)

흐름:
주소 입력 → HTTP/HTTPS로 요청 → 서버가 HTML 전송 → 브라우저가 렌더링
