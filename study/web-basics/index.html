개요

C++ 게임 서버 개발자가 웹 서버 코드를 읽기 전에 알아야 할 기초 개념을 정리했습니다. 브라우저와 웹서버의 차이, JavaScript와 Node.js의 관계를 명확하게 이해하는 것이 목표입니다.

1. 브라우저 vs 웹서버

인터넷의 기본 구조

당신 (사용자) 어딘가의 컴퓨터 (회사 서버실) │ │ ┌─────────┐ ┌─────────┐ │ 브라우저 │ ──── 인터넷 ────────────> │ 웹서버 │ │ (Chrome)│ <──────────────────────── │ │ └─────────┘ └─────────┘ "요청하는 쪽" "응답하는 쪽" (클라이언트) (서버)

브라우저란?

당신이 쓰는 프로그램. Chrome, Firefox, Edge, Safari 등.

  • 주소창에 URL 입력하면 → 서버에 요청 보냄
  • 서버에서 응답 받으면 → 화면에 예쁘게 그려줌

웹서버란?

어딘가에서 24시간 켜져있는 프로그램.

  • 요청이 올 때까지 기다림
  • 요청 오면 → 처리하고 → 응답 보냄
  • 다시 기다림

실제 예시: 네이버 접속

당신 네이버 서버실 │ │ │ 1. 브라우저에 "naver.com" 입력 │ │ │ │ 2. "네이버 메인페이지 주세요" ──────────> │ │ │ │ 3. 웹서버가 처리 │ - DB에서 뉴스 조회 │ - HTML 생성 │ │ │ 4. <html>네이버 메인</html> <────────── │ │ │ │ 5. 브라우저가 화면에 그림 │
구분 위치 역할
브라우저 당신 컴퓨터 요청 보내고, 결과 보여줌
웹서버 회사 서버 요청 받고, 처리해서 응답

2. JavaScript vs Node.js

JavaScript의 역사

1995년: 브라우저에서 태어남

웹 브라우저 (Netscape) │ └── "웹페이지에서 버튼 클릭하면 뭔가 반응하게 하고 싶다" │ └── JavaScript 탄생

원래 목적: 웹페이지에서 동적인 효과 (버튼 클릭, 팝업 등)

Node.js가 없던 시절 (2009년 이전)

JavaScript로 웹서버를 만들 수 없었다

JavaScript는 오직 브라우저 안에서만 실행 가능했다.

브라우저 안에서만 가능

  • 버튼 클릭하면 색 바꾸기
  • 팝업 띄우기
  • 입력값 검증하기

브라우저 밖에서는 불가능

  • 파일 읽기? X
  • DB 연결? X
  • 서버 만들기? X

그때 웹 개발은 어땠냐면

프론트엔드 (브라우저) 백엔드 (서버) │ │ JavaScript PHP, Java, Python 등 │ │ 버튼 클릭 효과 실제 데이터 처리

두 개의 다른 언어를 써야 했다.

2009년, Node.js 탄생

어떤 개발자의 생각

"JavaScript 엔진(V8)이 엄청 빨라졌네"
"이걸 브라우저 밖으로 꺼내면 어떨까?"
"파일도 읽고, 네트워크도 하고, 서버도 만들 수 있게?"

Node.js가 바꾼 것

Before: ┌─────────────────────┐ │ 브라우저 │ │ ┌───────────────┐ │ │ │ JavaScript │ │ ← 여기 갇혀있음 │ └───────────────┘ │ └─────────────────────┘ After (Node.js): ┌─────────────────────┐ │ 브라우저 │ │ ┌───────────────┐ │ │ │ JavaScript │ │ ← 여전히 가능 │ └───────────────┘ │ └─────────────────────┘ ┌─────────────────────┐ │ 서버/PC │ │ ┌───────────────┐ │ │ │ JavaScript │ │ ← 이제 여기서도 가능! │ │ (Node.js) │ │ │ └───────────────┘ │ └─────────────────────┘
구분 정체 역할
JavaScript 프로그래밍 언어 문법, 규칙 정의
브라우저 JavaScript 실행 환경 #1 웹페이지에서 실행
Node.js JavaScript 실행 환경 #2 서버에서 실행

3. 흔한 오해 바로잡기

표현 맞음?
"Node.js는 백엔드용 언어다" X
"Node.js는 JavaScript를 백엔드에서 실행하게 해주는 도구다" O
"Node.js로 개발한다" = "JavaScript로 서버 만든다" O

실무에서 이렇게 말한다

"이 서버 뭘로 만들었어요?" → "Node.js요" (JavaScript + Node.js라는 뜻)
"프론트는요?" → "React요" (이것도 JavaScript)

둘 다 JavaScript인데, 관례상 서버 쪽은 "Node.js로 만들었다"라고 말한다.

최종 요약

핵심 정리

JavaScript (언어) + Node.js (실행기) = 웹서버 개발 가능
