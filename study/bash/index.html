Bash: 터미널과 쉘의 기초

초보자를 위한 개념 정리 2025.01
Bash Shell 터미널 명령어 CLI

개요

개발하다 보면 검은 화면에 명령어를 치는 일이 많아요. ls, cd, grep 같은 명령어들...

"터미널", "콘솔", "쉘", "Bash"... 다 비슷해 보이는데 뭐가 뭔지, 그리고 자주 쓰는 명령어는 뭔지 정리해봅시다.

1. 컴퓨터에게 명령하는 두 가지 방법

질문

컴퓨터한테 명령을 내리는 방법이 뭐가 있을까요?

방법 설명 예시
GUI 마우스로 클릭 폴더 더블클릭해서 열기
CLI 글자로 명령 cd Documents 입력

CLI = Command Line Interface (명령 줄 인터페이스)

개발자들이 검은 화면에서 작업하는 게 바로 CLI 방식이에요!

2. 터미널, 콘솔, 쉘 - 뭐가 다른 거야?

핵심 구분

┌─────────────────────────────────────┐ │ 터미널 / 콘솔 (창) │ ← 화면 자체 │ ┌─────────────────────────────┐ │ │ │ Shell (쉘) │ │ ← 명령어 해석기 │ │ "ls", "cd" 등을 해석해서 │ │ │ │ 운영체제에 전달 │ │ │ └─────────────────────────────┘ │ └─────────────────────────────────────┘
용어 역할 비유
터미널/콘솔 검은 창 자체 (화면) TV 화면
Shell (쉘) 명령어를 해석하는 프로그램 TV 안의 방송국

쉘이 없으면?

ls 쳐도 컴퓨터가 이해를 못 해요!

쉘이 "ls = 파일 목록 보여줘"라고 통역해주는 거예요.

3. Bash란?

쉘에도 종류가 있어요. 마치 브라우저에 Chrome, Firefox, Edge가 있듯이!

OS 대표 쉘
Windows cmd, PowerShell
Linux / macOS Bash, zsh, sh

Bash = Bourne Again Shell

"옛날에 Bourne Shell이라는 게 있었는데, 그걸 다시(Again) 만들었다"는 뜻

리눅스/맥에서 가장 많이 쓰는 쉘이에요!

SSH와 연결

SSH로 리눅스 서버에 접속하면?

ssh user@server.com
user@server:~$    ← 이게 Bash 프롬프트!

바로 Bash 쉘이 떠요. 그래서 명령어를 칠 수 있는 거예요.

4. Git Bash는 뭐야?

질문

윈도우에서 Git Bash 써본 적 있나요?

Git Bash = 윈도우에서도 Bash처럼 쓸 수 있게 해주는 프로그램

윈도우인데... cmd에서: Git Bash에서: > dir $ ls ← 리눅스 명령어가 된다! > type file.txt $ cat file.txt > find "text" $ grep "text"

왜 Git Bash를 쓰나?

대부분의 개발 문서, 튜토리얼이 리눅스/Bash 기준으로 작성되어 있어요.

Git Bash를 쓰면 윈도우에서도 그대로 따라할 수 있어요!

5. 자주 쓰는 Bash 명령어

Claude Code, Cursor, VS Code 터미널에서도 이 명령어들을 사용해요!

파일/폴더 탐색

명령어 예시 설명
ls ls -la 파일/폴더 목록 보기 (-la: 상세 정보)
cd cd Documents 폴더 이동 (Change Directory)
pwd pwd 현재 위치 확인 (Print Working Directory)
find find . -name "*.js" 파일 찾기 (현재 폴더에서 .js 파일 찾기)

파일 조작

명령어 예시 설명
cat cat file.txt 파일 내용 전체 보기
head head -n 10 file.txt 파일 처음 10줄 보기
tail tail -n 10 file.txt 파일 마지막 10줄 보기
touch touch newfile.txt 빈 파일 생성
mkdir mkdir newfolder 폴더 생성
cp cp file.txt backup.txt 파일 복사
mv mv old.txt new.txt 파일 이동 또는 이름 변경
rm rm file.txt 파일 삭제 (주의! 휴지통 안 거침)

텍스트 검색

명령어 예시 설명
grep grep "error" log.txt 파일에서 "error" 포함된 줄 찾기
grep -r grep -r "TODO" ./src 폴더 전체에서 "TODO" 검색
grep -i grep -i "error" log.txt 대소문자 구분 없이 검색

기타 유용한 명령어

명령어 예시 설명
echo echo "Hello" 텍스트 출력
clear clear 화면 지우기
history history 이전에 친 명령어 목록
chmod chmod +x script.sh 파일 권한 변경 (실행 권한 추가)
| cat log.txt | grep "error" 파이프: 앞 명령의 결과를 뒤 명령에 전달
> echo "hello" > file.txt 결과를 파일로 저장 (덮어쓰기)
>> echo "world" >> file.txt 결과를 파일에 추가 (이어쓰기)

6. 실전 예시

프로젝트에서 특정 텍스트 찾기

# src 폴더에서 "useState" 사용한 곳 모두 찾기
grep -r "useState" ./src

# 결과 예시
./src/App.js:  const [count, setCount] = useState(0);
./src/Header.js:  const [isOpen, setIsOpen] = useState(false);

로그 파일 분석

# 로그 파일에서 에러만 보기
cat server.log | grep "ERROR"

# 마지막 50줄만 보기
tail -n 50 server.log

# 실시간으로 로그 보기 (새 로그가 추가되면 자동 출력)
tail -f server.log

파일 정리

# backup 폴더 만들고 파일 복사
mkdir backup
cp important.txt backup/

# 모든 .log 파일 삭제
rm *.log

7. Windows cmd vs Bash 명령어 비교

기능 Windows (cmd) Bash (Linux/Mac)
파일 목록 dir ls
폴더 이동 cd cd
현재 위치 cd (인자 없이) pwd
파일 내용 보기 type cat
파일 복사 copy cp
파일 이동 move mv
파일 삭제 del rm
폴더 생성 mkdir mkdir
화면 지우기 cls clear
텍스트 검색 find "text" grep "text"

최종 요약

Bash 한눈에 보기

터미널/콘솔 = 검은 창 (화면)

Shell (쉘) = 명령어를 해석하는 프로그램

Bash = 리눅스/맥의 대표 쉘 (Bourne Again Shell)

Git Bash = 윈도우에서 Bash처럼 쓸 수 있게 해주는 도구

핵심 명령어:
ls 목록 | cd 이동 | cat 보기 | grep 검색
cp 복사 | mv 이동 | rm 삭제 | mkdir 폴더생성
