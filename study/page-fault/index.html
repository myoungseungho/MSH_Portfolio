가상 메모리 & 페이지 폴트

RAM보다 큰 메모리 사용, 쓰래싱(Thrashing) 실측 테스트

개념: RAM보다 큰 메모리가 필요하면?

컴퓨터에 RAM이 16GB인데 20GB가 필요하면?

답: 디스크를 RAM처럼 사용한다 → 가상 메모리
[가상 메모리 구조] RAM (빠름) ─────────────────────┐ ┌─────────────────────────────┐ │ │ 자주 쓰는 데이터 │ │ └─────────────────────────────┘ │ ↕ 페이지 교환 │ 디스크 - 스왑 파일 (느림) ─────┘ ┌─────────────────────────────┐ │ 안 쓰는 데이터 임시 보관 │ └─────────────────────────────┘

페이지(Page)란?

메모리는 4KB 단위(페이지)로 관리된다.

  • RAM ↔ 디스크 교환도 페이지 단위
  • 안 쓰는 페이지 → 디스크로 내보냄
  • 필요한 페이지 → 디스크에서 RAM으로 올림

페이지 폴트 (Page Fault)

CPU가 메모리에 접근했는데 RAM에 없으면?

1. CPU: "메모리 주소 0x1234 읽어야지" 2. OS 확인: "어? 이거 RAM에 없네" → 페이지 폴트! 3. 디스크에서 해당 페이지 읽어서 RAM에 올림 4. 다시 실행
문제:
• RAM 접근: ~100 ns (나노초)
• 디스크 접근: ~100,000 ns (SSD 기준)
→ 페이지 폴트 = 1000배 느림

쓰래싱 (Thrashing)

페이지 폴트가 너무 자주 발생하면 시스템이 거의 멈춤.

쓰래싱: 프로그램이 일하는 시간보다 페이지 교환하는 시간이 더 많은 상태

테스트 1: 캐시 효과 (64MB)

RAM 안에서 순차 vs 랜덤 접근 비교:

[순차 접근] 64MB 배열, 10회 반복 결과: 272 ms [랜덤 접근] 64MB 배열, 10회 반복 결과: 3,528 ms [결과] 랜덤이 순차보다 13배 느림 → 캐시 미스로 인한 성능 저하

테스트 2: TLB 효과 (16MB)

같은 페이지 반복 vs 매번 다른 페이지 접근:

[같은 페이지 반복] 결과: 297 us [매번 다른 페이지] (스트라이드 4KB) 결과: 4,478 us [결과] 다른 페이지 접근이 15배 느림 → TLB 미스 + 캐시 미스
TLB (Translation Lookaside Buffer):
가상 주소 → 물리 주소 변환을 캐싱하는 버퍼.
TLB 미스 = 페이지 테이블 조회 필요 = 느림

테스트 3: 진짜 쓰래싱 (26GB)

31GB RAM 시스템에서 26GB 할당하여 스왑 유발:

[시스템 상태] 총 RAM: 31.1 GB 할당: 26 GB [초기화 후] 사용 가능 RAM: 0.01 GB ← RAM 바닥! 페이지 파일 사용: 56.4 GB ← 디스크 스왑 발생! [순차 접근] 10,000회 결과: 24 ms (1회당 2.4 us) [랜덤 접근] 10,000회 결과: 156 ms (1회당 15.6 us) [결과] 랜덤이 순차보다 6.5배 느림

왜 6.5배밖에 안 느릴까?

요인 설명
SSD가 빠름 HDD였으면 100배 이상 차이
Windows 최적화 프리페칭, 캐싱 잘 함
접근 횟수 10,000회는 SSD가 버틸 수 있음

메모리 접근 패턴별 속도

접근 패턴 속도 이유
순차 접근 (RAM) 매우 빠름 캐시 히트, 프리페치
랜덤 접근 (RAM) 느림 (10~15배) 캐시 미스, TLB 미스
순차 접근 (스왑) 느림 디스크 I/O, but 순차라 나음
랜덤 접근 (스왑) 매우 느림 디스크 랜덤 I/O = 최악

게임 서버에서의 교훈

주의사항:
• 메모리 사용량이 RAM에 근접하면 성능 급락
• 랜덤 메모리 접근 패턴 피하기
• 데이터 지역성(Locality) 고려한 설계 필요

실제 적용 예시

상황 나쁜 패턴 좋은 패턴
유저 데이터 순회 포인터 따라가며 랜덤 접근 배열에 연속 저장, 순차 순회
몬스터 AI 각 몬스터가 흩어진 메모리 몬스터 배열 연속 배치
아이템 검색 링크드 리스트 순회 해시맵 또는 연속 배열

핵심 정리

  • 가상 메모리 = RAM + 디스크 스왑으로 더 큰 메모리 사용
  • 페이지 폴트 = RAM에 없는 데이터 접근 시 디스크에서 로드
  • 쓰래싱 = 페이지 폴트 폭발 → 시스템 먹통
  • 순차 접근이 랜덤 접근보다 10~15배 빠름 (캐시/TLB)
  • SSD가 스왑 지옥을 완화하지만, RAM이 최고
  • 데이터 지역성 = 성능의 핵심
