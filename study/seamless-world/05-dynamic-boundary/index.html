Step 5: 월드 매니저가 부하에 따라 경계를 동적으로 조절

고정 경계의 한계

지금까지 배운 오버랩 존은 경계가 고정되어 있었습니다. 그런데 경계 근처에서 보스레이드가 벌어지면? 

존 A        |오버랩|        존 B
            | 100명 |
            |보스레이드|

오버랩에서 양쪽이 "둘 다 알고 있으면":
- 존 A: 100명 위치 추적
- 존 B: 100명 위치 추적
→ 비효율적!

경계가 고정되어 있어서 문제! 경계를 움직이면?

경계를 움직인다

[Before]
존 A 담당: X = 0 ~ 1000
존 B 담당: X = 900 ~ 2000
오버랩: 900 ~ 1000


[After - 경계를 존 B 쪽으로 밀면]
존 A 담당: X = 0 ~ 1200   ← 늘어남
존 B 담당: X = 1100 ~ 2000 ← 줄어듦
오버랩: 1100 ~ 1200       ← 위치가 옮겨짐

보스레이드가 있는 곳을 한 서버가 통째로 담당하면 중복 연산이 사라집니다!

월드 매니저 (상위 관리자)

경계를 "언제" "어디로" 밀지 누가 결정할까요? 

존 A, 존 B가 직접 대화하면?

존 A: "나 지금 부하 80%야, 경계 좀 밀어줘"
존 B: "나도 70%인데? 너가 더 받아"
존 A: "아니 너가 받아"
→ 서로 싸움!
상위 관리자가 있으면?

        [월드 매니저]  ← 전체 상황 모니터링
         /          \
      존 A          존 B

월드 매니저: "존 A 80%, 존 B 40%네"
월드 매니저: "존 B야, 경계 니 쪽으로 밀어서 좀 더 받아"
존 B: "넵"
월드 매니저 역할설명
부하 모니터링"존 A 몇 %, 존 B 몇 %"
경계 이동 결정"경계를 어디로 밀어라"
유저 배치 조율"이 유저는 존 B로 보내"

경계 이동 과정

핵심: 끊기지 않게!

경계를 "확" 밀면 유저가 갑자기 끊길 수 있음. 그래서:

  1. 오버랩 먼저 확장 (안전하게 양쪽 연결)
  2. 유저 핸드오프 (정보 전달)
  3. Authority 이전
  4. 옛 영역 해제
경계 이동 순서:

Step 1: 오버랩을 먼저 넓힘
        [존 A    |====오버랩 확장====|    존 B]

Step 2: 새 영역의 유저들 핸드오프
        존 B → 존 A: "이 유저들 정보야"

Step 3: Authority 이전
        새 영역 유저들의 Authority가 존 A로

Step 4: 오버랩 축소, 옛 영역 해제
        [존 A         |오버랩|    존 B]

→ "오버랩 확장 → 이전 → 축소" 순서!

존 기반 심리스 최종 정리

전체 구조

        [월드 매니저]
         ↙    ↓    ↘
      존 A   존 B   존 C ...

각 존 서버:
- 담당 영역의 유저 관리
- 인접 존과 오버랩 영역 공유
- 핸드오프로 유저 정보 교환

월드 매니저:
- 전체 부하 모니터링
- 동적 경계 조절 명령
- 유저 배치 최적화

존 기반 심리스 핵심 정리

[기초 심리스]
- 오버랩 존에서 양쪽 서버 동시 연결
- Authority 기준으로 판정 (가까운 쪽)
- 핸드오프로 유저 정보 전달
- 넘어가면 이전 서버 해제

[동적 심리스]
- 월드 매니저가 전체 상황 모니터링
- 부하에 따라 경계 동적 조절
- 오버랩 확장 → 이전 → 축소 순서
