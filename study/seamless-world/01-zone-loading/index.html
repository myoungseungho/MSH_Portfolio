Step 1: 존 이동 시 로딩 화면이 발생하는 구조

왜 로딩 화면이 뜰까?

게임에서 한 지역에서 다른 지역으로 이동할 때 "로딩 중..." 화면을 본 적 있죠? 왜 그런 걸까요? 

[존 A 서버] ──── 경계 ──── [존 B 서버]

유저가 경계를 넘으면:
1. 존 A 서버에서 접속 해제
2. 존 B 서버에 새로 접속
3. 존 B 서버가 맵 데이터 로드
4. 유저 입장 완료

→ 이 과정에서 "로딩 중..." 화면

핵심은 "끊고 → 다시 연결" 하기 때문입니다.

기존 방식의 문제

시간 흐름 →

[유저가 존 A에 있음]
    ↓
[경계에 도달]
    ↓
[존 A 연결 해제] ← 여기서 끊김!
    ↓
[로딩 화면...]   ← 유저는 기다림
    ↓
[존 B에 연결]
    ↓
[맵 데이터 로드]
    ↓
[입장 완료]

유저 입장에서는 "걷다가 갑자기 멈추고 로딩 화면을 봐야 하는" 경험입니다. 몰입감이 깨지죠.

로딩이 생기는 이유 정리

  • 연결 끊김: 기존 서버와 연결을 먼저 끊음
  • 재연결: 새 서버에 처음부터 연결 시작
  • 데이터 로드: 새 맵 데이터를 받아와야 함

이 세 과정 동안 유저는 아무것도 할 수 없음 → 로딩 화면

그럼 어떻게 해결할까?

로딩이 생기는 이유가 "끊고 다시 연결"이라면...

"끊기 전에 미리 연결해두면?" 

[기존: 끊고 연결]
존 A 접속 중 → 끊음 → (로딩) → 존 B 연결

[심리스: 미리 연결]
존 A 접속 중 → 존 B에 미리 연결 → 존 A 해제
              ↑
         이 순간 양쪽 다 연결된 상태!

이게 바로 심리스(Seamless)의 핵심 원리입니다!

핵심 포인트

이번 Step에서 배운 것
