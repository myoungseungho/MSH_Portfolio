Step 2: 오버랩 존 (Overlap Zone)

단계: 2/5 난이도: 기초
Overlap Dual Connection Boundary

Step 2: 오버랩 존에서 양쪽 서버에 동시 연결

오버랩 존이란?

"미리 연결해둔다"는 게 구체적으로 어떻게 동작할까요? 바로 오버랩 존(Overlap Zone)을 통해서입니다. 

   존 A 서버          경계          존 B 서버
[=============]|◀─ 오버랩 ─▶|[=============]

유저가 오버랩 진입:
1. 존 A에 연결된 상태에서
2. 존 B에도 "미리" 연결
3. 양쪽 서버가 동시에 이 유저를 앎
4. 완전히 넘어가면 존 A에서 해제

유저 입장에선 그냥 걸어가는 것뿐. 끊김 없이 자연스럽게!

오버랩 존은 별도 서버가 아니다!

많이 헷갈리는 부분인데, 오버랩 존은 별도의 프로세스가 아닙니다. 

[잘못된 이해]
존 A 서버     오버랩 서버     존 B 서버
   □      ─      □      ─      □
                 ↑
            별도 프로세스? ❌ 아님!


[실제 구조]
존 A 서버                     존 B 서버
   □ ─────────────────────────── □

   [존 A 담당 영역  |겹침|  존 B 담당 영역]
                    ↑
               그냥 "영역"일 뿐
               둘 다 이 영역을 알고 있음

좌표로 이해하기

오버랩 존 = 그냥 좌표 범위

존 A 서버 설정: 내 담당 구역 X = 0 ~ 1100
존 B 서버 설정: 내 담당 구역 X = 900 ~ 2000

겹치는 부분 (900 ~ 1100) = 오버랩 존

별도 서버가 아니라, 두 서버가 같은 영역을 동시에 담당하는 것!

유저가 오버랩에 들어오면?

유저 위치: X = 950 (오버랩 존 안)

존 A 서버: "950은 내 범위(0~1100)니까 내가 처리"
존 B 서버: "950은 내 범위(900~2000)니까 나도 처리"

→ 둘 다 이 유저를 알고 있음
→ 단, 실제 연산은 한쪽만 함 (Authority)

핵심 포인트

이번 Step에서 배운 것

  • 오버랩 존: 양쪽 서버가 동시에 담당하는 영역
  • 별도 서버가 아닌 좌표 범위가 겹치는 것
  • 유저가 오버랩 진입 시 → 양쪽 서버에 동시 연결
  • 완전히 넘어가면 → 이전 서버에서 해제
