🤖 하루 24시간 사냥하는 유저를 어떻게 잡을까?

게임 매크로/봇 탐지와 방어 전략

1️⃣ 매크로가 뭔데?

게임에서 "매크로 쓴다"는 게 무슨 뜻이야?

게임을 자동으로 플레이하는 프로그램이야.

사람이 키보드/마우스를 누르는 대신, 프로그램이 대신 눌러주는 거지.

예를 들면?

MMORPG에서 몬스터 사냥을 생각해봐:

1. 몬스터 찾기 2. 클릭해서 공격 3. 스킬 버튼 누르기 4. 아이템 줍기 5. 1번으로 돌아가기

이걸 24시간 내내 반복하는 거야. 사람은 자는 동안에도.

🎮 비유: 자동 청소 로봇

룸바(로봇 청소기)를 생각해봐.
사람이 빗자루로 청소하는 대신, 로봇이 알아서 움직이며 청소하지?

게임 매크로도 똑같아.
사람이 게임하는 대신, 프로그램이 알아서 게임해.

2️⃣ 그럼 편한데, 왜 문제야?

매크로 쓰면 편한데 왜 금지해?

3가지 이유가 있어:

❌ 문제 1: 경제 파괴

매크로 유저가 24시간 사냥 → 아이템 무한 생산 → 아이템 가격 폭락

진짜 유저들이 열심히 모은 아이템의 가치가 사라져.

❌ 문제 2: 불공정 경쟁

A 유저: 하루 2시간 플레이 → 레벨 50

B 유저 (매크로): 하루 24시간 플레이 → 레벨 200

경쟁이 안 돼. 게임이 재미없어져.

❌ 문제 3: 서버 부하

매크로 1000명이 24시간 돌리면 = 사람 12000명이 플레이하는 효과

서버가 터져.

그럼 게임 회사는 어떻게 해?

계정 영구 정지 (BAN)

리니지, 메이플스토리, 로스트아크 등 대부분의 게임이 매크로를 금지하고, 적발 시 계정을 차단해.

3️⃣ 매크로랑 봇이랑 다른 거야?

사람들이 "매크로" "봇" 이렇게 섞어 쓰던데, 같은 거야?

비슷하지만 복잡도가 다르다고 보면 돼.

매크로 (Macro) 봇 (Bot)
단순 반복 작업 복잡한 AI 판단
"1초마다 F1 키 누르기" "몬스터 위치 인식 → 이동 → 공격"
AutoHotkey, 키보드 매크로 이미지 인식 + 경로 찾기
예: 자동 물약 먹기 예: 자동 사냥 (이동 포함)
Q. 그럼 봇이 더 위험한 거네?

맞아. 봇은 사람처럼 행동해서 탐지하기 어려워.

매크로는 패턴이 단순해서 잡기 쉽지만, 봇은 랜덤 요소를 넣어서 사람처럼 위장해.

4️⃣ 매크로는 어떻게 만들어?

프로그래밍 몰라도 만들 수 있어?

너무 쉬워. 그래서 문제야.

방법 1: 키보드 매크로 프로그램

AutoHotkey, Pulover's Macro Creator 같은 프로그램 다운로드 ↓ "녹화" 버튼 누르고 게임 플레이 ↓ "정지" 누르면 내 행동이 저장됨 ↓ "반복" 버튼 누르면 계속 재생

코딩 필요 없음. 3분이면 만들 수 있어.

방법 2: 마우스 매크로

마우스 클릭 위치를 픽셀 좌표로 저장:

1. (500, 300) 클릭 - 몬스터 위치 2. 1초 대기 3. (700, 500) 클릭 - 스킬 버튼 4. 2초 대기 5. 1번으로 돌아가기

문제: 해상도 바뀌면 안 돼. 몬스터 위치 바뀌면 안 돼.

방법 3: 이미지 인식 봇 (고급)

OpenCV 같은 라이브러리로 화면 인식:

while (true) { cv::Mat screen = CaptureScreen(); cv::Point monsterPos = FindImage(screen, "몬스터.png"); if (monsterPos.x != -1) { MoveToPosition(monsterPos); ClickAttackButton(); } if (currentHP < maxHP * 0.3f) { UsePotion(); } }

이제 진짜 봇이야. 화면을 "보고" 판단해.

5️⃣ 게임 회사는 어떻게 잡아?

매크로 쓰는 사람이랑 진짜 사람을 어떻게 구분해?

사람과 프로그램의 행동 패턴 차이를 찾아.

🔍 탐지 방법들

1️⃣ 반복 패턴 탐지

사람의 사냥 패턴: A 몬스터 잡기 (15초) → B 몬스터 (18초) → 휴식 (5초) → A 몬스터 (14초) → 시간이 불규칙적 매크로의 사냥 패턴: A 몬스터 잡기 (15초) → B 몬스터 (15초) → A 몬스터 (15초) → B 몬스터 (15초) → 시간이 정확히 똑같음 ⚠️

해결책: 서버에서 행동 간격을 측정 → 편차가 0에 가까우면 의심

2️⃣ 마우스 움직임 분석

사람: 마우스가 곡선으로 움직임 클릭 전에 약간 떨림 속도가 가변적 매크로: 마우스가 직선으로 이동 (A → B 최단거리) 클릭이 픽셀 단위로 정확함 속도가 일정함

해결책: 마우스 궤적 데이터 수집 → 머신러닝으로 패턴 학습

3️⃣ 키 입력 타이밍

사람이 "스킬1 → 스킬2 → 스킬3" 누를 때: 0.00초 - 스킬1 0.73초 - 스킬2 (0.73초 간격) 1.51초 - 스킬3 (0.78초 간격) 매크로: 0.00초 - 스킬1 0.50초 - 스킬2 (0.50초 간격) 1.00초 - 스킬3 (0.50초 간격) ⚠️

정확히 0.5초 간격이면 매크로 확정.

4️⃣ 플레이 시간 분석

정상 유저: 하루 평균 3시간 플레이 주로 저녁 7~11시 주말에 더 많이 플레이 매크로 유저: 하루 평균 20시간 플레이 ⚠️ 새벽 3시에도 플레이 중 ⚠️ 휴식 없이 계속 사냥 ⚠️

해결책: 연속 플레이 6시간 이상 → 경고 / 12시간 이상 → 조사

5️⃣ CAPTCHA (캡차)

"나는 로봇이 아닙니다" 체크박스 또는 이미지 선택:

무작위 타이밍에 팝업: "신호등이 있는 이미지를 모두 선택하세요" 사람: 클릭해서 통과 매크로: 응답 없음 → 자동 차단

문제: 게임 플레이 중 캡차가 뜨면 짜증남. 일반 유저 불편.

6️⃣ 클라이언트 무결성 검사

게임 실행 파일이 변조되었는지 체크: 원본 파일 해시값: ABC123DEF456 현재 파일 해시값: ABC123DEF456 ✅ 메모리에 이상한 프로그램이 있는지 체크: - AutoHotkey.exe 실행 중? ⚠️ - Cheat Engine 실행 중? ⚠️

대표 프로그램: EasyAntiCheat, BattlEye, XIGNCODE

7️⃣ 머신러닝 기반 탐지

정상 유저 10만 명 데이터 학습 ↓ 패턴 모델 생성 ↓ 실시간으로 의심 유저 점수 계산 ↓ 점수 90점 이상 → 관리자 검토 → BAN

예: 블리자드의 Warden, 라이엇의 Vanguard

6️⃣ 완벽하게 막을 수는 없어?

이 정도면 매크로 완전히 막을 수 있는 거 아냐?

불가능해.

왜냐하면 매크로 제작자들도 계속 진화하거든.

🔴 공격자의 대응

문제: 패턴이 똑같아서 걸림 해결: 랜덤 대기 시간 추가 문제: 마우스가 직선으로 움직임 해결: 베지어 곡선으로 움직임 문제: 플레이 시간 너무 김 해결: 2시간마다 10분 쉬기 문제: CAPTCHA 뜸 해결: 사람이 풀어주는 서비스 이용 (1회당 $0.001)

🟢 방어자의 대응

문제: 랜덤 딜레이 추가함 해결: 랜덤 패턴도 학습 가능 머신러닝으로 "가짜 랜덤" 탐지 문제: 곡선 움직임 사용 해결: 사람의 곡선과 프로그램의 곡선은 미묘하게 다름 문제: 주기적으로 쉼 해결: 행동 효율성 측정 (너무 완벽 = 의심)
Q. 그럼 매크로와의 싸움은 끝이 없다는 거네?

정확해!

이건 보안 전쟁이야. 한쪽이 막으면 다른 쪽이 뚫고, 또 막고...

게임 회사는 계속 돈을 써서 안티치트 팀을 운영해야 해.

7️⃣ 실제 게임은 어떻게 막아?

🎮 사례 1: 블리자드 (WoW, 오버워치)

전략: Warden 시스템 + 신고 시스템

1. 클라이언트에 Warden 상주 (메모리 감시) 2. 의심 행동 자동 감지 → 서버 전송 3. 유저 신고 → GM(관리자) 검토 4. 확정 시 계정 영구 정지 + 하드웨어 ID 차단

결과: 봇 사용자가 새 계정 만들어도 같은 컴퓨터면 차단

🎮 사례 2: 리니지M (모바일)

전략: AI 기반 패턴 분석

1. 유저 행동 데이터 실시간 수집 2. AI가 "의심 점수" 계산 3. 점수 높은 유저 → 게임 내 미니게임 띄우기 (예: 간단한 퍼즐 풀기) 4. 실패 or 미응답 → 임시 정지 → 관리자 검토

논란: 일반 유저도 가끔 걸림. 오탐 문제.

🎮 사례 3: 로스트아크

전략: 다층 방어

Layer 1: EasyAntiCheat (안티치트 프로그램) Layer 2: 서버 측 행동 패턴 분석 Layer 3: 골드 거래 추적 (RMT 방지) Layer 4: 유저 신고 시스템 Layer 5: 대규모 BAN 웨이브 (한 번에 수만 명)

전략: 조금씩 잡지 않고 증거 모아서 한 번에 대량 차단

Q. 왜 한 번에 대량으로 BAN해?

매크로 제작자들이 어떤 패턴에서 걸렸는지 학습하지 못하게 하려고.

만약 한 명씩 바로바로 BAN하면:

"아, A 패턴 쓰면 걸리네? 그럼 B 패턴으로 바꿔야지" → 빠르게 대응 가능

하지만 3개월 동안 모았다가 한 번에 BAN하면:

"뭘로 걸린 거야? A? B? C?" → 정확한 원인 파악 어려움

8️⃣ 내가 게임 서버 개발자라면?

실제로 서버에서 매크로 탐지 코드를 어떻게 짜?

간단한 예시를 보자:

예시: 행동 간격 측정

// 유저가 몬스터를 잡을 때마다 호출 void OnMonsterKill(int userId, int64_t timestamp) { int64_t prevTime = GetUserLastKillTime(userId); double interval = timestamp - prevTime; // 간격을 기록 AddInterval(userId, interval); // 최근 100번의 간격 분석 std::vector intervals = GetRecentIntervals(userId, 100); double stdDev = CalculateStdDev(intervals); // 표준편차가 너무 작으면 의심 if (stdDev < 0.1) { FlagUser(userId, "매크로 의심"); } }

표준편차가 0에 가까우면 = 간격이 너무 일정 = 매크로 가능성 높음

예시: 플레이 시간 체크

// 1시간마다 실행 void HourlyCheck() { std::vector users = GetOnlineUsers(); for (const auto& user : users) { int playTime = GetContinuousPlayTime(user.id); if (playTime > 6 * 60 * 60) { // 6시간 SendWarning(user.id, "장시간 플레이 중입니다"); } if (playTime > 12 * 60 * 60) { // 12시간 ShowCaptcha(user.id); } if (playTime > 18 * 60 * 60) { // 18시간 FlagForReview(user.id); } } }

예시: 이동 경로 검증

// 유저가 이동할 때마다 void OnPlayerMove(int userId, Vector2 fromPos, Vector2 toPos, int64_t timestamp) { float distance = CalculateDistance(fromPos, toPos); int64_t timeDiff = timestamp - GetLastMoveTime(userId); float speed = distance / timeDiff; float maxSpeed = GetPlayerMaxSpeed(userId); if (speed > maxSpeed * 1.2f) { // 20% 여유 KickUser(userId, "스피드핵 감지"); return; } // 경로가 직선인지 체크 if (IsStraightLine(fromPos, toPos) && HasObstacle(fromPos, toPos)) { FlagUser(userId, "벽 통과 의심"); } }

📌 핵심 정리: 매크로/봇 탐지

  • 매크로 - 단순 반복 작업 자동화 (키보드 매크로)
  • 봇 - 복잡한 AI 판단 포함 (이미지 인식 등)
  • 문제점 - 경제 파괴, 불공정 경쟁, 서버 부하
  • 탐지 방법 - 패턴 분석, 타이밍 측정, 머신러닝, CAPTCHA
  • 현실 - 완벽한 방어 불가능. 계속되는 보안 전쟁
  • 전략 - 다층 방어 + 대규모 BAN 웨이브로 학습 방지
  • 서버 측 검증 - 클라이언트는 믿지 말고 서버에서 재검증

최종 결론

그럼 매크로는 절대 못 막는 거야?

100% 방어는 불가능해.

하지만 비용을 높일 수는 있어:

매크로 제작 비용 ↑ 탐지 확률 ↑ BAN 시 손실 ↑ (계정 + 하드웨어 차단) → "이거 쓰느니 그냥 정상적으로 하는 게 낫네"

목표는 완벽한 차단이 아니라 매크로를 쓰기 어렵게 만드는 것.
