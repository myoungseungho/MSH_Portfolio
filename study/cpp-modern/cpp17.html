C++17 Features

Modern C++ 실용적 기능 대거 추가
C++17 structured binding optional variant string_view

개요

C++17은 실용적인 기능이 많이 추가되었습니다. 특히 코드 간결화와 타입 안전성 향상에 좋은 것들이 많아요.

1구조적 바인딩

질문
pair에서 값을 꺼낼 때 .first, .second 매번 쓰기 귀찮은데... 
std::pair<int, std::string> getUser() {
    return {42, "Kim"};
}

// C++14: 귀찮게 꺼냄
auto result = getUser();
int id = result.first;
std::string name = result.second;
구조적 바인딩!
// C++17: 한 줄로 분해
auto [id, name] = getUser();

// id = 42
// name = "Kim"

map 순회할 때 더 편함

std::map<int, std::string> users = {{1, "Kim"}, {2, "Lee"}};

// C++14
for (auto& p : users) {
    std::cout << p.first << ": " << p.second;  // 귀찮
}

// C++17
for (auto& [id, name] : users) {
    std::cout << id << ": " << name;  // 깔끔!
}

2if문 초기화

질문
if 안에서만 쓸 변수인데, 밖에서도 보이는 게 싫다 
// C++14: it이 스코프 밖에서도 살아있음
auto it = users.find(42);
if (it != users.end()) {
    // it 사용
}
// 여기서도 it이 보임...
if 안에서 변수 선언!
// C++17: 스코프 제한
if (auto it = users.find(42); it != users.end()) {
    // it 사용
}
// 여기선 it 안 보임!

// switch도 가능
switch (auto val = getValue(); val) {
    case 1: ...
    case 2: ...
}

세미콜론(;)으로 초기화와 조건을 분리합니다.

3std::optional

질문
값이 없을 수도 있는 걸 어떻게 표현하지? -1? nullptr? 
// 기존 방식들의 문제
int findAge(int id) {
    if (못찾으면)
        return -1;  // -1이 진짜 나이일 수도 있음!
}

Player* findPlayer(int id) {
    if (못찾으면)
        return nullptr;  // 포인터 강제됨
}
"없을 수도 있음"을 타입으로!
#include <optional>

std::optional<int> findAge(int id) {
    if (못찾으면)
        return std::nullopt;  // 명확하게 "없음"
    return 25;
}

// 사용
auto age = findAge(42);
if (age) {  // 있으면
    std::cout << *age;
}

// 기본값 지정
int a = age.value_or(0);  // 없으면 0

optional의 진짜 가치

함수 시그니처만 봐도 의도를 알 수 있음:

int getAge();                    // 항상 반환한다고 생각함
std::optional<int> getAge();     // "없을 수도 있구나"

4std::variant

질문
여러 타입 중 하나를 담고 싶은데, union은 위험하고... 
// C 스타일 union: 위험
union { int n; char* s; } u;
u.n = 42;
char* str = u.s;  // 잘못 읽어도 컴파일 됨... 런타임 터짐
타입 안전한 variant!
#include <variant>

// int 또는 string 또는 double 중 하나
std::variant<int, std::string, double> data;

data = 42;          // int 저장
data = "hello";     // string 저장
data = 3.14;        // double 저장

// 현재 뭐가 들어있는지 확인
if (std::holds_alternative<int>(data)) {
    int n = std::get<int>(data);
}

// 잘못된 타입으로 꺼내면 예외 발생 (안전!)
std::string s = std::get<std::string>(data);  // 예외!

5if constexpr

질문
템플릿에서 타입에 따라 다른 코드를 실행하고 싶은데...

템플릿의 특징

템플릿은 코드 틀입니다. 호출할 때 타입을 넣어서 실제 코드가 생성됨:

template<typename T>
void process(T value) {
    value++;       // T=int면 OK
    value.size();  // T=string이면 OK
}

process(42);  // T=int로 코드 생성하려는데...
              // int에 .size() 없으니까 컴파일 에러!

문제: 일반 if문은 양쪽 다 컴파일하려고 함

컴파일 타임 분기!
template<typename T>
void process(T value) {
    if constexpr (std::is_integral_v<T>) {
        value++;      // T가 정수일 때만 컴파일
    } else {
        value.size(); // T가 정수 아닐 때만 컴파일
    }
}

if constexpr은 조건 안 맞는 쪽을 아예 컴파일 안 함.

6std::string_view

질문
const char*도, string도 복사 없이 받고 싶은데... 
void process(const std::string& str);

process("hello");  // const char* → std::string 임시 객체 생성 (복사!)
읽기 전용 뷰!
#include <string_view>

void process(std::string_view str) {
    // str 사용
}

process("hello");           // const char* → OK, 복사 없음
process(std::string("hi")); // std::string → OK, 복사 없음

string_view는 포인터 + 길이만 들고 있는 "읽기 전용 뷰"

주의: 원본이 살아있어야 함

std::string_view dangerous() {
    std::string temp = "hello";
    return temp;  // temp 사라지면 쓰레기 참조!
}

소유권 없이 빌려보는 것뿐. 원본 수명 주의!
