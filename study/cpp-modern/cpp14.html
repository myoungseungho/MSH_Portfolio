C++14 Features

Modern C++ C++11 편의성 개선
C++14 make_unique generic lambda auto return

개요

C++14는 C++11의 편의성 개선이 주된 내용입니다. 큰 변화보다는 "이게 왜 C++11에 없었지?" 싶은 것들이 추가되었습니다.

1std::make_unique

질문
shared_ptr에는 make_shared가 있는데, unique_ptr은 왜 new를 직접 써야 할까? 
// C++11
std::unique_ptr<Player> p1 = std::unique_ptr<Player>(new Player());

// shared_ptr은 편한데...
std::shared_ptr<Player> p2 = std::make_shared<Player>();
생각해보면...

make_unique가 있어야 하는 거 아닌가?

C++14에서 추가!
// C++14: make_unique 추가
std::unique_ptr<Player> p = std::make_unique<Player>();

make_unique가 C++11에서 빠진 건 실수였다고 합니다.

2제네릭 람다

질문
람다마다 타입을 다 지정해야 하나? 여러 타입을 받고 싶은데... 
// C++11: 타입 고정
auto add = [](int a, int b) { return a + b; };

add(1, 2);      // OK
add(1.5, 2.5);  // 정수로 변환됨...
생각해보면...

매개변수에 auto를 쓸 수 있으면?

제네릭 람다!
// C++14: 제네릭 람다
auto add = [](auto a, auto b) { return a + b; };

add(1, 2);        // int
add(1.5, 2.5);    // double
add(str1, str2);  // string 연결도 가능!

내부적으로 템플릿처럼 동작합니다.

3auto 반환 타입

질문
반환하는 게 뻔한데, 왜 반환 타입을 또 써야 하지? 
// C++11: 반환 타입 명시 필요
auto getData() -> std::vector<std::map<std::string, int>> {
    std::vector<std::map<std::string, int>> result;
    return result;
}
생각해보면...

그냥 auto만 쓰면 안 되나?

auto 반환 타입 추론!
// C++14: 반환 타입 자동 추론
auto getData() {
    std::vector<std::map<std::string, int>> result;
    return result;  // 컴파일러가 타입 추론
}

주의: 모든 return이 같은 타입이어야 함

auto getValue(bool flag) {
    if (flag)
        return 1;    // int
    else
        return 2.0;  // double - 에러! 타입 불일치
}

4숫자 구분자

질문
0이 몇 개인지 세기 힘든데... 
long long budget = 1000000000000;  // 이게 얼마야?
작은따옴표로 구분!
// C++14: 숫자 구분자
long long budget = 1'000'000'000'000;  // 1조

int binary = 0b1010'1010;  // 이진수도 가능
int hex = 0xFF'AA'BB'CC;   // 16진수도 가능

컴파일러는 '를 무시합니다. 순수하게 가독성을 위한 기능.

5[[deprecated]]

질문
더 이상 안 쓰는 함수인데, 쓰면 경고를 주고 싶다면?
[[deprecated]] 속성!
// C++14: deprecated 표시
[[deprecated]]
void oldFunction() {
    // ...
}

[[deprecated("newFunction()을 사용하세요")]]
void oldFunction2() {
    // ...
}

// 사용하면 컴파일 경고 발생
oldFunction();  // warning: 'oldFunction' is deprecated

팀 프로젝트에서 레거시 코드 정리할 때 유용합니다.
