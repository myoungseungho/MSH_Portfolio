방화벽과 포트: 서버 보안의 기초

Network Server
Firewall Port RDP SSH FTP SMB

시작: 이상한 상황

어느 날 겪은 일

클라우드 서버에서 30GB짜리 파일을 내 PC로 복사해야 했습니다.

평소처럼 원격 데스크톱(mRemoteNG)으로 서버에 접속했습니다. 잘 됩니다. 화면도 보이고, 마우스도 움직입니다.

그런데 파일을 복사하려고 하니까...

시도 1: robocopy (Windows 파일 복사 명령어) → "연결 시간 초과" 시도 2: scp (보안 파일 복사 명령어) → "연결 시간 초과" 시도 3: ftp (파일 전송 프로토콜) → "연결 시간 초과"

🤔 이상하지 않나요?

원격 데스크톱은 되는데, 파일 복사는 왜 안 될까요?
둘 다 같은 서버에 접속하는 건데...

이 질문의 답을 찾으려면, 포트와 방화벽이 뭔지 알아야 합니다.

1단계: "포트"가 뭔가요?

먼저 질문 하나

인터넷으로 뭔가 접속한다고 생각해보세요.

  • 웹사이트 볼 때 → 브라우저 사용
  • 게임할 때 → 게임 클라이언트 사용
  • 원격 접속할 때 → mRemoteNG 사용

🤔 생각해보기

이것들이 다 같은 "문"으로 서버에 들어갈까요?
아니면 각각 다른 "문"으로 들어갈까요?

정답: 각각 다른 문으로 들어갑니다

서버는 건물이라고 생각하면 됩니다.
이 건물에는 문이 여러 개 있어요. 이 문 하나하나가 포트입니다.

서버 = 건물 ┌──────────────────────────────────────────┐ │ 서버 │ │ │ │ 🚪 3389번 문 - 원격 데스크톱 전용 │ │ 🚪 22번 문 - 명령어/파일전송 전용 │ │ 🚪 21번 문 - 파일전송 전용 │ │ 🚪 80번 문 - 웹사이트 전용 │ │ 🚪 ... │ │ │ └──────────────────────────────────────────┘

핵심 개념

포트 = 서버의 문
각 문은 번호가 있고, 각각 다른 용도로 사용됩니다.

자주 쓰는 포트 번호

전 세계가 약속한 "이 번호는 이 용도로 쓰자"라는 규칙이 있어요.

포트 번호 이름 용도 비유
3389 RDP 원격 데스크톱 건물에 직접 들어가서 일하기
22 SSH 명령어로 서버 조작 전화로 지시하기
22 SFTP 파일 전송 (보안 좋음) 보안 택배
21 FTP 파일 전송 (보안 낮음) 일반 택배
445 SMB Windows 폴더 공유 옆집 문 열고 들어가기
80 HTTP 웹사이트 가게 입구
포트 번호는 바꿀 수 있나요?

네, 바꿀 수 있어요!

위 번호들은 "약속"일 뿐, 강제가 아닙니다.

예시: 기본: SSH는 22번 변경: SSH를 2222번으로 바꿔서 운영

왜 바꾸나요?

해커들이 22번 포트를 먼저 공격하니까,
다른 번호로 숨기는 거예요. (보안 목적)

비유

편의점은 보통 1층에 있어요.
하지만 5층에 편의점을 만들어도 됩니다.
찾기 어려울 뿐, 가능은 해요.

2단계: "방화벽"이 뭔가요?

문이 있다고 다 들어갈 수 있을까요?

서버에 문(포트)이 여러 개 있다는 건 알았어요.
그런데 문이 있다고 아무나 들어갈 수 있을까요?

🤔 생각해보기

실제 건물에도 문이 있지만, 아무나 못 들어가잖아요.
왜요? 경비원이 있으니까요!

방화벽 = 경비원

방화벽은 각 문(포트) 앞에 서 있는 경비원이에요.

서버 건물 ┌─ 🚪 3389번 문 ─ 👮 경비원: "들어오세요" │ 서버 ───┼─ 🚪 22번 문 ─── 👮 경비원: "출입 금지!" │ └─ 🚪 445번 문 ── 👮 경비원: "출입 금지!"

핵심 개념

방화벽 = 경비원
각 문(포트)마다 "들여보낼지 말지"를 결정합니다.

경비원(방화벽)이 하는 일

경비원의 지시 예시: • "3389번 문은 누구나 들여보내" • "22번 문은 아무도 들여보내지 마" • "445번 문은 특정 IP만 들여보내" • "80번 문은 누구나 들여보내"

경비원이 2명?!

여기서 중요한 게 있어요. 실제로는 경비원이 2명입니다.

인터넷에서 서버로 가는 길: 나 → [👮 경비원1: 클라우드 방화벽] → [👮 경비원2: 서버 방화벽] → 서버 • 경비원1: 클라우드 회사에서 관리 (NCP, AWS 콘솔에서 설정) • 경비원2: 서버 안에서 관리 (Windows 방화벽, Linux iptables)

중요!

경비원1(클라우드 방화벽)에서 막히면, 서버까지 아예 도착을 못 해요.
서버 안에서 아무리 설정해도 소용없습니다.

3단계: 처음 질문의 답

다시 처음으로 돌아가서...

원격 데스크톱(RDP)은 되는데, 파일 복사(robocopy, scp, ftp)는 왜 안 됐을까요?

각각 사용하는 포트가 달랐습니다

시도한 것 사용하는 포트 경비원(방화벽) 상태 결과
mRemoteNG (원격 데스크톱) 3389 ✅ 열림 됨!
robocopy (파일 복사) 445 ❌ 막힘 연결 시간 초과
scp (보안 파일 복사) 22 ❌ 막힘 연결 시간 초과
ftp (파일 전송) 21 ❌ 막힘 연결 시간 초과

원인 정리

클라우드 방화벽(경비원1)이 3389번 문만 열어놓고,
나머지 문(22, 21, 445)은 다 막아놓은 상태였습니다.

그래서 원격 데스크톱(3389번 문)은 됐지만,
파일 복사(다른 문)는 안 됐던 거예요.

그림으로 보면

나 ──→ 클라우드 방화벽 ──→ 서버 3389번 문으로 가면? 나 → 👮 "어서오세요" → 🚪 → 서버 접속 성공! 22번 문으로 가면? 나 → 👮 "출입 금지!" → ❌ (연결 시간 초과) 445번 문으로 가면? 나 → 👮 "출입 금지!" → ❌ (연결 시간 초과)

4단계: 어떻게 해결하나요?

방법 1: 방화벽에서 포트 열기

클라우드 콘솔에서 필요한 포트를 열어달라고 요청합니다.

NCP(네이버 클라우드) 예시: 콘솔 → Server → ACG (Access Control Group) → 인바운드 규칙 추가: | 프로토콜 | 포트 | 허용 대상 | |----------|------|-----------| | TCP | 22 | 내 IP만 | | TCP | 445 | 내 IP만 |

보안 팁

포트를 열 때는 "내 IP만" 허용하는 게 안전해요.
"모든 IP 허용"으로 하면 해커도 들어올 수 있습니다.

방법 2: 열려있는 포트로 우회하기

만약 방화벽 설정을 바꿀 수 없다면?
이미 열려있는 포트(RDP)를 활용합니다.

예: 30GB 파일을 옮겨야 할 때 1. 원격 데스크톱(3389)으로 서버 접속 2. 서버 안에서 파일을 1GB씩 쪼갬 3. 쪼갠 파일을 하나씩 복사 (Ctrl+C, V) 4. 내 PC에서 다시 합침

방법 3: 다른 전송 방법 사용

예전에 FileZilla로 됐다면, 그때 사용한 포트가 열려있는 거예요.

FileZilla 설정 확인: • 포트 21로 됐다면 → FTP(21)가 열려있음 • 포트 22로 됐다면 → SFTP(22)가 열려있음 • 다른 포트였다면 → 그 포트가 특별히 열려있음

자주 쓰는 포트 (외워두면 좋아요)

포트 용도 언제 쓰나요?
3389 RDP (원격 데스크톱) Windows 서버 화면 보면서 조작할 때
22 SSH / SFTP 명령어로 조작하거나, 파일 보안 전송할 때
21 FTP 파일 전송할 때 (보안 낮음)
445 SMB (파일 공유) \\IP\폴더 로 접근할 때
80 / 443 HTTP / HTTPS 웹사이트 접속할 때

핵심 교훈

  • 서버 접속이 안 되면 → "어떤 포트로 접속하는 건지?" 먼저 확인
  • 그 포트가 방화벽에서 열려있는지 확인
  • 클라우드 방화벽 + 서버 방화벽, 2단계 모두 확인 필요
