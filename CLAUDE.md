# CLAUDE.md - MSH Portfolio 작성 가이드

이 문서는 Claude Code가 MSH Portfolio 프로젝트에서 학습 문서를 작성할 때 따라야 할 지침입니다.

---

## 📝 문서 작성 원칙

### 1. 스캐폴딩 방식 (Scaffolding)

**독자가 스스로 깨달을 수 있도록 단계적 질문으로 유도합니다.**

```
❌ 나쁜 예: "AOI는 관심 영역입니다. 서버 부하를 줄이기 위해 사용합니다."

✅ 좋은 예:
Q. MMO에서 내가 서울에 있는데, 부산에 있는 유저 정보까지 받아야 해?
A. 필요 없지. 어차피 안 보이는데 왜 받아?
   그래서 나온 개념이 AOI (Area of Interest) = 관심 영역
```

**핵심: 답을 바로 주지 말고, 질문으로 생각을 유도한다.**

---

### 2. 대화형 Q&A 구조

모든 문서는 **대화 형식**으로 작성합니다:

```html
<div class="chat">
    <div class="q">왜 이게 필요해?</div>
    <div class="a">
        <p>설명...</p>
    </div>
</div>
```

**Q (질문)**: 독자가 자연스럽게 가질 만한 의문
**A (답변)**: 간결하고 명확한 설명

---

### 3. 시각 자료 (GIF/이미지) 필수

개념 설명 시 **단계별 시각 자료**를 포함합니다:

```
images/
├── step1.gif  - 문제 상황 시각화
├── step2.gif  - 첫 번째 해결 시도
├── step3.gif  - 개선된 방법
├── step4.gif  - 최적화
└── step5.gif  - 최종 결과
```

**GIF 제작 원칙:**
- 단순한 도형/색상으로 개념 표현
- 한 GIF당 하나의 개념만 설명
- 3~5초 내 핵심 전달

```html
<div class="visual-section">
    <h4>시뮬레이션: 문제 상황</h4>
    <img src="./images/step1.gif" alt="Step 1" class="step-gif">
    <p class="gif-caption">설명 캡션</p>
</div>
```

---

### 4. 하이라이트 시스템

중요도에 따라 다른 스타일 사용:

| 클래스 | 용도 | 색상 |
|--------|------|------|
| `.highlight` | 핵심 개념/용어 | 파란색 |
| `.key-point` | 핵심 포인트/결론 | 초록색 |
| `.warning` | 주의/경고/오해 | 빨간색 |

```html
<span class="highlight">Googlebot</span>  <!-- 핵심 개념 -->
<span class="key-point">정확해!</span>     <!-- 결론 -->
<span class="warning">불가능해.</span>     <!-- 주의 -->
```

---

### 5. 비유 활용

추상적 개념은 **일상적 비유**로 설명:

```html
<div class="analogy-box">
    <strong>🏪 비유: 가게 오픈</strong><br><br>
    당신이 서울 한복판에 가게를 열었어...
</div>
```

**좋은 비유의 조건:**
- 독자가 이미 아는 것과 연결
- 핵심 차이점을 명확히 드러냄
- 과도한 확장 금지 (비유는 비유일 뿐)

---

### 6. 비교 테이블

두 개 이상의 개념 비교 시:

```html
<table class="compare-table">
    <tr>
        <th>개념 A</th>
        <th>개념 B</th>
    </tr>
    <tr>
        <td>특징 1</td>
        <td>특징 1</td>
    </tr>
</table>
```

---

### 7. 핵심 정리 박스

문서 끝에 **핵심 요약** 필수:

```html
<div class="summary-box">
    <h3>📌 핵심 정리: 제목</h3>
    <ul>
        <li><strong>키워드</strong> - 한 줄 설명</li>
        ...
    </ul>
</div>
```
